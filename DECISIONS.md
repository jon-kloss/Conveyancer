# DECISIONS

One line per judgment call beyond the docs — what, why, which doc section it touches. Review surface for the human.

- **Workspace layout:** `solver` crate holds both T0 and T1; `solver-wasm` is a thin cdylib wrapper over T0 for wasm-pack (SDD §2 names "solver-core → WASM"; a separate wrapper keeps good_lp/LP deps out of the wasm build). — SDD §2
- **Renderer directory:** React app lives in `renderer/` (not `app/`) to avoid colliding with the `app` crate name. — SDD §2
- **Dev bridge:** `app` crate ships two entry points — the Tauri shell and a `dev-bridge` HTTP+SSE binary exposing the identical command layer, so the renderer can be driven and screenshot in a headless container (no display) against the *real* Rust core. Tauri deps are feature-gated (`shell`, on by default; CI container builds it too now that WebKitGTK is present). Rust still owns canonical state in both modes; no state forking. — SDD §4, HANDOFF "verify as you go"
- **Map imagery:** no community tile set is bundled yet — licensing must be verified before ship (SDD §7). Phase 1 renders the documented placeholder treatment (diagonal-stripe = runtime asset placeholder) over the survey grid + biome ghost labels. **OPEN QUESTION for human: which community tile set / license to pursue.** — SDD §7, UI spec Assets
- **World-node snapshot:** bundled `world-nodes.json` is a hand-assembled subset (iron/copper/limestone/coal in the Grass Fields / Rocky Desert / Northern Forest areas, real-ish coordinates, correct purity distribution shape) sufficient for Phase 1 planning. Full community-data snapshot swap-in is tracked in BACKLOG (v1). — SDD §3 "static world data"
- **Docs.json fixture:** repo bundles a trimmed, UTF-8 Docs.json-format fixture covering the Modular Frame chain (+miners/belts/power) so tests and the dev app run without a game install; the parser detects UTF-16LE (real installs) vs UTF-8 (fixture) by BOM. — SDD §7
- **Internal factory wiring:** SDD §3 models inter-factory `Route`s but not intra-factory connections; the graph view needs them. Added `BeltEdge { factory, from, to, item, tier }` entity (endpoints = machine group or boundary port); flow rates on edges are derived solver output, tier is user-set. Same ◇◈◆ + flow grammar as routes. — SDD §3, UI spec screen 6
- **T0 count semantics:** "scaling counts/clocks/rates linearly" (SDD §5.1) implemented as: required group throughput scales linearly; count = ceil(throughput / machine base rate) with clock redistributed evenly (counts/clocks are numbers, not geometry, so Principle 5 holds). T1 does the same relax-round-redistribute per SDD §5.2.
